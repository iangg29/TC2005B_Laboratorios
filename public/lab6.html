Preguntas

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es una buena práctica ya que se pueden desplegar errores o ayudas al usuario para que pueda llenar los inputs de una manera correcta.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Existen diferentes formas, por ejemplo apagar los scripts dentro del navegador, inyectar nuevo código, así como poder hacer overwrite del mismo para saltar validaciones.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Es una buena práctica, ya que normalmente las validaciones se realizan en ambos lados de la aplicación (frontend y backend), por lo que las validaciones con JavaScript son importantes para la retroalimentación al usuario y no tanto de seguridad.
